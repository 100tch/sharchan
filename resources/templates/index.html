Automate your process in the daily life

sharchan provides features that will help you with the transfer of information and automation of any processes.

What is this?

sharchan is a simple and insecure file hosting service created as part of an educational project. It is mainly intended for transferring small files using HTTP requests, which means you can automate some things by making requests through scripts.

Features

Open Source

Our solution being open source does not represent an advantage; it just means that users can not only use this service but also modify the source code and deploy it for personal use. The source code is available under the BSD 2-Clause license.

Simple API

We provide a simple and intuitive API that allows for easy integration of our solution into your workflows. With our API, you can automate tasks effectively.

Link Shortening

With our tool, you can easily shorten long links, making them more convenient for use and sharing. This is especially useful for social media, email newsletters, and other platforms where character limits apply.

No Ads

We value your time and attention, which is why our solution is completely ad-free. You can focus on your tasks without distractions.

How it Work?

λ curl -F 'file=@helloworld.txt' {{domain}} 
/f/a948904f2f0f479b8f8197694b30184b0d2ed1c1cd2a1ec0fb85d299a192a447
λ curl {{domain}}/f/a948904f2f0f479b8f8197694b30184b0d2ed1c1cd2a1ec0fb85d299a192a447
hello world

Using the first command we sent a POST request to upload the file `helloworld.txt` to the server. After that, we using the second command to retrived the contents of that file.

Note: The output from second command can be redirected to a new file by using the following syntax:
curl {{domain}}/f/%some-hash% > file.txt

Instead of sending a POST request using the `-F` parameter, you can use the `curl -T` command to send a PUT request. This command allows you to upload a file to the server using the HTTP PUT method.

λ curl -T yourfile.jpg {{domain}} | tee
λ command | curl -T - {{domain}}

You can also pipe input from another command directly into `curl` like this. In this case, the hyphen `-` signifies that `curl` should read from standard input. This way you can upload command output.

Note: You can also upload the output of a command, including any error messages from `stderr`, to the server using the following syntax:

command 2>&1 | curl -T - {{domain}}

In this command, `2>&1` redirects `stderr` (file descriptor 2) to `stdout` (file descriptor 1), allowing both standard output and standard error to be sent through the pipe.

Any of the three command will help you shorten the link and then use the shortened version to access the file. This can be useful when distributing the link.

λ curl -F 'url=http://{{domain}}/%some-long-url%' http://{{domain}}/shorten
λ curl -d 'url=http://{{domain}}/%some-long-url%' http://{{domain}}/shorten
λ curl -X POST http://{{domain}}/shorten -d 'url=http://{{domain}}/%some-long-url%'
