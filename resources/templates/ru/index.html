Автоматизируй процессы своей повседневной жизни

sharchan предоставляет функции, которые помогут вам с передачей информации и автоматизацией любых процессов.

Что это такое?

sharchan — это простой и небезопасный файлообменник, созданный в рамках образовательного проекта. В основном он предназначен для передачи небольших файлов с помощью HTTP-запросов, что означает, что вы можете автоматизировать некоторые вещи, делая запросы через скрипты.

Особенности

Открытый Код

То что наше решение является открытым не является преимуществом; это просто означает, что пользователи могут не только использовать сервис, но и изменять исходный код и развертывать его для личного использования. Исходный код доступен по лицензии BSD 2-Clause.

Простой API

Мы предоставляем простой и интуитивно понятный API, который позволяет легко интегрировать наше решение в ваши рабочие процессы. С нашим API вы можете эффективно автоматизировать задачи.

Сокращение Ссылок

С помощью нашего инструмента вы можете легко сократить длинные ссылки, сделав их более удобными для использования и распространения. Это особенно полезно для социальных сетей, электронных рассылок и других платформ, где действуют ограничения на количество символов.

Нет Рекламы

Мы ценим ваше время и внимание, поэтому наше решение полностью лишено рекламы. Вы можете сосредоточиться на своих задачах, не отвлекаясь.

Как это работает?

λ curl -F 'file=@helloworld.txt' localhost:3000
/f/a948904f2f0f479b8f8197694b30184b0d2ed1c1cd2a1ec0fb85d299a192a447
λ curl localhost:3000/f/a948904f2f0f479b8f8197694b30184b0d2ed1c1cd2a1ec0fb85d299a192a447
hello world

С помощью первой команды мы отправили POST-запрос на загрузку файла `helloworld.txt` на сервер. После этого мы использовали вторую команду для извлечения содержимого этого файла.

Примечание: Вывод второй команды можно перенаправить в новый файл, используя следующий синтаксис:
curl localhost:3000/f/%some-hash% > file.txt

Вместо отправки запроса POST с помощью параметра `-F`, вы можете использовать команду `curl -T` для отправки запроса PUT. Эта команда позволяет вам загрузить файл на сервер с помощью метода HTTP PUT.

λ curl -T yourfile.jpg localhost:3000 | tee
λ command | curl -T - http://localhost:3000

Вы также можете передать входные данные из другой команды напрямую в `curl`, как здесь. В этом случае дефис `-` означает, что `curl` должен считывать данные со стандартного ввода. Таким образом вы можете загрузить выходные данные команды.

Примечание: Вы также можете загрузить вывод команды, включая любые сообщения об ошибках из `stderr`, на сервер, используя следующий синтаксис:

command 2>&1 | curl -T - http://localhost:3000

В этой команде `2>&1` перенаправляет `stderr` (файловый дескриптор 2) в `std out` (файловый дескриптор 1), позволяя отправлять через канал как стандартный вывод, так и стандартную ошибку.

Любая из трех команд поможет вам сократить ссылку и затем использовать сокращенную версию для доступа к файлу. Это может быть полезно при распространении ссылки.

λ curl -F 'url=http://localhost/%some-long-url%' http://localhost:3000/shorten
λ curl -d 'url=http://localhost/%some-long-url%' http://localhost:3000/shorten
λ curl -X POST http://localhost:3000/shorten -d 'url=http://localhost/%some-long-url%'
